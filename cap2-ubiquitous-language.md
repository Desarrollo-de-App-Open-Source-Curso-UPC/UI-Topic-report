
## 2.4. Ubiquitous Language

En esta sección se presentan los términos clave del proyecto UI-Topic. Estos términos han sido acordados en el equipo del proyecto y serán usados de manera consistente en la comunicación, documentación y desarrollo del sistema.

| Término                  | Definición                                                                 |
|--------------------------|----------------------------------------------------------------------------|
| Restaurante              | Establecimiento comercial que necesita gestionar su inventario.           |
| Inventario               | Conjunto de productos e insumos utilizados por el restaurante.            |
| Insumo                   | Ingrediente o producto utilizado para la preparación de platos.           |
| Producto                 | Elemento que puede ser parte del inventario. Puede ser consumible o no.   |
| Proveedor                | Persona o entidad que entrega productos o insumos al restaurante.         |
| Stock                    | Cantidad actual disponible de un insumo o producto.                       |
| Pedido                   | Solicitud de reposición de insumos o productos al proveedor.              |
| Usuario Administrador    | Persona responsable de gestionar el sistema de inventario.                |
| Alerta                   | Notificación automática cuando un insumo está por agotarse.               |
| Panel de control         | Interfaz donde el administrador visualiza y gestiona todo el sistema.     |
| API                      | Interfaz de programación que permite la conexión entre frontend y backend.|
| Landing Page             | Página informativa principal del proyecto.                                |
| Aplicación Web           | Sistema interactivo que permite al usuario gestionar el inventario.       |
